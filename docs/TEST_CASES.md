# 智能视频混剪工作台测试用例

## 1. 测试目标

本文档基于 `PRD.md`、当前 v0.1-v0.4 实现和标准迭代验收流程编写，目标是覆盖智能视频混剪工作台的全部功能场景、异常场景、边界场景和核心非功能要求。

测试范围分为三类：

- 当前可执行：已在现有产品中实现或已有接口/页面入口。
- 条件可执行：需要特定环境，例如 ffmpeg、Whisper 命令、真实 LLM endpoint。
- 未来预留：PRD P2 或待确认能力，当前用于产品验收追踪，不要求现版本通过。

## 2. 测试环境与基础数据

### 2.1 推荐环境

- 后端：`cd backend && uv run uvicorn app.main:app --host 127.0.0.1 --port 8000`
- 前端：`cd frontend && npm run dev -- --port 5173`
- 浏览器：Codex in-app browser，地址 `http://127.0.0.1:5173/`
- 自动化检查：
  - `cd backend && uv run pytest`
  - `cd frontend && npm run build`

### 2.2 测试素材

- `valid_01.mp4`：可播放 MP4，5-30 秒，含基础 metadata。
- `valid_02.mov`：可上传 MOV。
- `valid_03.webm`：可上传 WEBM。
- `valid_04.mkv`：可上传 MKV。
- `valid_05.avi`：可上传 AVI。
- `invalid.txt`：非法格式。
- `broken.mp4`：损坏视频。
- `no_audio.mp4`：无音频视频。
- `large.mp4`：超过 `VIDEO_CUT_MAX_VIDEO_BYTES`。
- `long_name_<200 chars>.mp4`：超长文件名。
- `<script>alert(1)</script>.mp4`：注入型文件名。

### 2.3 测试账号与数据约束

当前版本无登录、单机单租户。测试时可直接通过 UI 或 API 创建工作空间。涉及真实 AI 的用例必须通过环境变量配置：

- 配置真实 pipeline 所需的 LLM、Whisper、ffmpeg
- `VIDEO_CUT_LLM_ENDPOINT`
- `VIDEO_CUT_LLM_API_KEY`
- `VIDEO_CUT_LLM_MODEL`
- `VIDEO_CUT_WHISPER_COMMAND`

API key 不允许写入代码、截图、日志或提交记录。
